# Generated Darling dependency bundle

This folder is generated by the GitHub Actions workflow:

- `.github/workflows/build-darling-dependencies.yml`
- `scripts/build_darling_dependency_bundle.sh`

The bundle targets Ubuntu 22.04 amd64 and is intended for testing/building Darling on Linux.

## Files

- `darling_offline_bundle_ubuntu_22_04_amd64.zip` if the ZIP is small enough for GitHub.
- `darling_offline_bundle_ubuntu_22_04_amd64.zip.part-*` if the ZIP was split to stay below GitHub's per-file size limit.
- `darling_offline_bundle_ubuntu_22_04_amd64.zip.sha256` contains the SHA-256 hash of the original ZIP.

## Reassemble split parts

If split parts were committed, run this from the repository root:

```bash
cat generated/darling_offline_bundle_ubuntu_22_04_amd64.zip.part-* > darling_offline_bundle_ubuntu_22_04_amd64.zip
sha256sum -c generated/darling_offline_bundle_ubuntu_22_04_amd64.zip.sha256
```

The Actions run also uploads the original ZIP as a workflow artifact, but the repository copy under `generated/` is the durable copy.
